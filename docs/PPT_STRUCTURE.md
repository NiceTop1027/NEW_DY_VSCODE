# 🏫 DY Web IDE - 최종 발표 자료 목차

## Session 1. Intro (프로젝트 소개)
*   **1.1 오프닝**: "코딩 교육, 설치하다 시간 다 보내실 건가요?"
*   **1.2 프로젝트 개요**: 설치 없이 브라우저로 접속하는 올인원 코딩 교실, DY Web IDE
*   **1.3 팀 소개**: 개발팀 역할 및 소개

## Session 2. Problem & Solution (문제와 해결)
*   **2.1 우리가 겪은 불편함**: 
    *   실습실 컴퓨터 포맷 때마다 사라지는 설정
    *   집에서는 환경이 달라 실습 불가 (OS 호환성)
    *   "선생님, 저 실행이 안 돼요" → 수업 흐름 20분 끊김
*   **2.2 우리의 해결책**:
    *   **No Install**: 접속 즉시 준비 완료 (5초)
    *   **Any Device**: 크롬북, 태블릿, 구형 노트북 모두 OK
    *   **Continuity**: 학교에서 하던 거 집에서 그대로

## Session 3. Core Features (핵심 기능)
*   **3.1 VS Code 경험 그대로**: 현업 표준 UI(Monaco Editor)를 채택하여 이질감 제로
*   **3.2 강력한 언어 지원**: C언어, Python, JS를 서버 없이 브라우저 자체 구동
*   **3.3 GitHub 과제 제출**: 복잡한 git 명령어 몰라도 클릭 한 번으로 수행평가 제출
*   **3.4 실시간 미리보기**: HTML/CSS 코드를 치는 순간 옆에 결과가 뜸

## Session 4. Challenge & Overcoming (개발 과정의 도전)
> *단순한 기술 나열이 아닌, 문제 해결 과정을 스토리텔링합니다.*

*   **4.1 난관 봉착**: "브라우저에서 파이썬과 C언어를 어떻게 돌리지?"
    *   **해결**: WebAssembly 기술 도입, 서버 부하 없이 클라이언트에서 처리
*   **4.2 파일 시스템 문제**: "웹은 파일 저장이 안 되는데?"
    *   **해결**: 브라우저 내부 저장소(IndexedDB)를 활용한 가상 파일 시스템 구현
*   **4.3 성능 최적화**: "너무 느리면 학생들이 안 쓴다"
    *   **해결**: 에디터와 실행 엔진 분리 (Web Worker) 및 모듈 경량화

## Session 5. Live Demo Scenario (시연 계획)
> *백 번의 설명보다 한 번의 시연이 강력합니다.*

*   **Scene 1. 접속과 시작**: URL 접속 → 로딩 2초 → 바로 "Hello World" 출력
*   **Scene 2. 알고리즘 풀이**: Python으로 별 찍기 예제 작성 및 실행 속도 확인
*   **Scene 3. 웹 사이트 제작**: HTML 태그 입력 시 실시간으로 화면 바뀌는 모습 (Wow 포인트)
*   **Scene 4. 과제 제출**: 'GitHub 로그인' → '제출하기' 버튼 클릭 → 선생님 화면에서 확인

## Session 6. Impact & Vision (기대 효과 및 비전)
*   **6.1 정량적 효과**: 
    *   수업 준비 시간: 20분 → **10초**
    *   실습 참여율 증가 및 이탈률 감소
*   **6.2 정성적 가치**: 
    *   도구 설치에 에너지를 쏟지 않고, **'코드 로직'** 그 자체에 집중하는 진짜 교육
*   **6.3 최종 비전**:
    *   덕영고를 넘어, 대한민국 모든 SW 교육 현장의 표준 플랫폼으로 도약

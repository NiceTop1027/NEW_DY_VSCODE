# 📄 프로젝트 상세 명세서 (포트폴리오용)

## 1. 프로젝트 개요

*   **프로젝트명**: DY Web IDE (덕영고등학교 웹 통합 개발 환경)
*   **프로젝트 기획 배경**:
    *   학교 실습실의 컴퓨터 환경이 매번 초기화되어, 수업 때마다 개발 환경을 다시 설정해야 하는 번거로움 존재.
    *   OS(Windows, Mac) 간의 차이로 인해 학생들의 실습 환경 불일치 문제 발생.
    *   별도의 설치 없이 웹 브라우저만으로 코딩 실습이 가능한 환경 필요.
*   **프로젝트 목표**:
    *   사용자가 URL 접속만으로 5초 이내에 개발 환경(VS Code UI)을 구성.
    *   서버 사이드 실행(Docker)과 클라이언트 실행(WebAssembly)을 하이브리드로 지원하여 성능 최적화.
    *   GitHub 연동을 통해 과제 제출 및 프로젝트 관리를 자동화.
*   **주요 기능**:
    *   **웹 기반 코드 에디터**: Monaco Editor를 사용하여 VS Code와 동일한 UX 제공.
    *   **다언어 런타임**: Python, C, C++, JavaScript 실행 지원 (Docker & WASM).
    *   **가상 파일 시스템**: 브라우저 내에 폴더/파일 생성 및 관리 (IndexedDB).
    *   **클라우드 동기화**: GitHub OAuth를 통한 레포지토리 연동.
*   **필요성**:
    *   개발 환경 설정의 진입 장벽을 제거하여 학생들이 코딩 로직 학습에만 집중할 수 있게 함.
    *   학교와 집 어디서든 연속적인 학습 경험 제공.

## 3. 서비스 소개

*   **서비스 소개**:
    *   **DY Web IDE**는 설치가 필요 없는 클라우드 기반 코딩 실습실입니다. 복잡한 환경 설정 없이 브라우저(Chrome, Edge 등)만 있으면 C언어, Python, JS 등을 즉시 작성하고 실행할 수 있습니다. 특히 학교 실습 환경에 최적화되어 있어, 선생님은 URL 공유만으로 수업 준비를 끝낼 수 있고, 학생은 자신의 코드를 GitHub에 안전하게 저장할 수 있습니다.
*   **서비스 대표 이미지**:
    *   `[메인 에디터 실행 화면 스크린샷]`

## 4. 개발 환경 및 기술 스택

*   **개발 언어 및 프레임워크**:
    *   **JavaScript (ES6+)**:
        *   (비교군: React/Vue) - **도입 이유**: 에디터 특성상 DOM 조작이 빈번하고 성능이 중요하여, 프레임워크 오버헤드를 제거하기 위해 Vanilla JS를 사용.
    *   **Node.js & Express**:
        *   (비교군: Django/Spring) - **도입 이유**: WebSocket(Socket.io/ws)을 통한 실시간 터미널 통신 구현에 유리하며, I/O 처리가 뛰어남.
*   **데이터베이스 & 스토리지**:
    *   **IndexedDB (BrowserFS)**:
        *   **도입 이유**: 사용자 코드 파일을 브라우저 로컬에 영구 저장하여 서버 스토리지 비용 절감.
    *   **GitHub API**:
        *   **도입 이유**: 별도의 DB 서버 구축 없이, 사용자의 GitHub 레포지토리를 데이터베이스처럼 활용하여 관리 부담 최소화.
*   **서버 환경 및 격리**:
    *   **Docker Container**:
        *   **도입 이유**: 사용자 코드가 실행될 때 시스템을 격리하여 보안 사고(악성 코드 실행 등)를 원천 차단.
*   **빌드 및 배포 도구**:
    *   **Webpack**:
        *   **도입 이유**: Monaco Editor의 무거운 리소스를 번들링하고 최적화하기 위해 사용.
    *   **Railway**:
        *   **도입 이유**: Dockerfile 기반 배포가 간편하고 SSL 인증서 자동 적용 등 인프라 관리 용이.

## 6. 설계 및 아키텍처

*   **시스템 아키텍처**:
    *   `Frontend (Browser)` <-> `WebSocket` <-> `Backend (Node.js)` <-> `Docker Container (Ubuntu User Session)`
*   **데이터 흐름**:
    1.  사용자가 코드 작성 (Monaco Editor)
    2.  '실행' 버튼 클릭 -> WebSocket으로 소스코드 전송
    3.  Node.js 서버가 Docker 컨테이너 내부에 파일 생성
    4.  컨테이너 내부에서 컴파일/실행 (`python main.py`)
    5.  실행 출력값(stdout)을 WebSocket으로 프론트엔드 터미널(Xterm.js)에 전송
*   **UI/UX 디자인 가이드**:
    *   VS Code의 "Dark+" 테마를 벤치마킹하여 개발자 친화적인 색상 및 레이아웃 채택.

## 7. 주요 기능 상세 설명

*   **1. Monaco Editor 기반 코드 편집**:
    *   VS Code와 동일한 구문 강조(Syntax Highlighting), 자동 완성(Intellisense), 미니맵 기능 제공.
*   **2. 샌드박스 터미널 (Sandbox Terminal)**:
    *   웹 브라우저에서 `ls`, `mkdir`, `python` 등의 리눅스 명령어를 그대로 사용할 수 있는 실제 쉘 환경 제공. (Docker 격리)
*   **3. GitHub 통합 (Sync)**:
    *   `git clone`, `git push` 과정을 UI 버튼으로 간소화하여, 명령어 없이도 프로젝트 버전 관리 가능.
*   **4. 지능형 파일 트리**:
    *   Drag & Drop 파일 업로드, 파일 생성/삭제/이름바꾸기 등 데스크탑 탐색기와 유사한 UX 제공.

## 8. 테스트/QA/피드백

*   **테스트 종류**:
    *   **Unit Test**: Jest를 사용하여 파일 시스템 유틸리티 함수의 정확성 검증.
    *   **Integration Test**: WebSocket 연결 및 Docker 컨테이너 생성/삭제 시나리오 테스트.
*   **보안 점검 (Security QA)**:
    *   Path Traversal 공격 방어 (`../../etc/passwd` 접근 차단 로직) 테스트 완료.
    *   Fork Bomb 방지 등 컨테이너 자원 제한 테스트 완료.

## 9. 배포 및 운영

*   **프로젝트 실행 방법**:
    ```bash
    git clone https://github.com/NiceTop1027/DY_VScode.git
    npm install
    npm run start  # 개발 서버 시작
    ```
*   **배포 프로세스**:
    *   GitHub `main` 브랜치 푸시 -> Railway Webhook 감지 -> Docker Image 빌드 -> 자동 배포(CD).
*   **운영 전략**:
    *   비용 절감을 위해 유휴 상태인 컨테이너는 10분 후 자동 종료(Garbage Collection) 로직 적용.
